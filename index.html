Đồng bộ điện thoại & máy tính · miễn phí
💡 Supabase là gì?
Tủ lưu trữ trên mây miễn phí. Thêm từ trên điện thoại buổi sáng, mở máy tính buổi tối vẫn còn đó. Không cần Supabase vẫn dùng được bình thường.
!
Trạng thái
Đang dùng LocalStorage · Dữ liệu lưu trong trình duyệt này
1
Tạo tài khoản & project
1. Vào supabase.com → Sign up miễn phí
2. New project → đặt tên → region Asia
3. Settings → API → copy URL và anon key
2
Tạo bảng (1 lần)
Vào SQL Editor trong Supabase, paste và chạy:
-- 1. Từ vựng CREATE TABLE IF NOT EXISTS words ( id BIGSERIAL PRIMARY KEY, word TEXT NOT NULL, phonetic TEXT DEFAULT '', part_of_speech TEXT DEFAULT '', meaning TEXT NOT NULL, example TEXT DEFAULT '', topic TEXT DEFAULT 'general', level INT DEFAULT 3, created_at TIMESTAMPTZ DEFAULT now() ); ALTER TABLE words ENABLE ROW LEVEL SECURITY; CREATE POLICY "public_all" ON words FOR ALL USING (true) WITH CHECK (true); -- 2. Câu ngữ pháp CREATE TABLE IF NOT EXISTS sentences ( id BIGSERIAL PRIMARY KEY, viet TEXT NOT NULL DEFAULT '', eng TEXT NOT NULL DEFAULT '', note TEXT DEFAULT '', type TEXT DEFAULT 'daily', created_at TIMESTAMPTZ DEFAULT now() ); ALTER TABLE sentences ENABLE ROW LEVEL SECURITY; CREATE POLICY "public_all" ON sentences FOR ALL USING (true) WITH CHECK (true); -- 3. Meta (tiến độ, goal, hard words, families) CREATE TABLE IF NOT EXISTS user_meta ( key TEXT PRIMARY KEY, value TEXT ); ALTER TABLE user_meta ENABLE ROW LEVEL SECURITY; CREATE POLICY "public_all" ON user_meta FOR ALL USING (true) WITH CHECK (true);
3

GitHub & Deploy
Hướng dẫn đưa app lên GitHub Pages — miễn phí, chạy online vĩnh viễn
💡 Tại sao cần GitHub?
Hiện tại app chỉ chạy trên máy bạn. Đưa lên GitHub Pages = có đường link riêng, mở được trên bất kỳ thiết bị nào, không mất tiền hosting, data vẫn lưu trong trình duyệt (hoặc Supabase).
1
Tạo tài khoản GitHub
Truy cập github.com → Sign up → Xác nhận email.
Nếu đã có tài khoản, bỏ qua bước này.
2
Tạo Repository mới
1. Nhấn nút + (góc trên phải) → New repository
2. Đặt tên: lingo-app (hoặc tên bất kỳ)
3. Chọn Public (bắt buộc để dùng GitHub Pages miễn phí)
4. Tick Add a README file
5. Nhấn Create repository
3
Upload file HTML
1. Trong repository vừa tạo, nhấn Add file → Upload files
2. Kéo file lingo-v4.html vào hoặc nhấn "choose your files"
3. Quan trọng: Đổi tên file thành index.html (GitHub Pages cần tên này)
4. Kéo xuống, nhấn Commit changes
Tên file phải là: index.html (không phải lingo-v4.html hay bất kỳ tên khác)
4
Bật GitHub Pages
1. Vào tab Settings của repository
2. Tìm mục Pages (trong thanh sidebar trái)
3. Mục Source → chọn Deploy from a branch
4. Branch: chọn main → Folder: / (root)
5. Nhấn Save
6. Chờ 1–2 phút, làm mới trang
5
Truy cập app của bạn
Sau khi deploy xong, link app sẽ có dạng:
https://[username].github.io/[repo-name]/ Ví dụ: https://nguyenvan.github.io/lingo-app/
Link này bạn có thể:
✓ Bookmark trên điện thoại (Add to Home Screen)
✓ Chia sẻ cho người khác
✓ Mở trên bất kỳ thiết bị nào
6
Cập nhật app khi có phiên bản mới
Khi muốn cập nhật, làm lại bước 3: vào repository → index.html → nhấn icon bút chì (Edit) → paste nội dung mới → Commit. GitHub Pages sẽ tự cập nhật sau 1–2 phút.
⚠ Lưu ý về dữ liệu
Data (từ vựng, tiến độ) lưu trong localStorage của trình duyệt đó. Nếu học trên điện thoại và máy tính khác nhau, data sẽ không tự đồng bộ. Để đồng bộ đa thiết bị, cần kết nối Supabase (xem tab riêng trong sidebar).
Supabase — Đồng bộ dữ liệu
Kết nối cloud database · Sync đa thiết bị · Hoàn toàn miễn phí
💡 Tại sao cần Supabase?
Không có Supabase, data chỉ lưu trong trình duyệt — đổi điện thoại hoặc xóa cache là mất hết. Với Supabase, mọi thứ lưu trên cloud: từ vựng, tiến độ, Word Family, câu ngữ pháp — học trên điện thoại, mở máy tính vẫn còn.
!
Trạng thái kết nối
Chưa kết nối · Đang dùng LocalStorage
1
Lấy URL và Key từ Supabase
1. Vào supabase.com → Sign up miễn phí (dùng GitHub hoặc email)
2. Nhấn New project → đặt tên → chọn region Southeast Asia
3. Vào Project Settings → API → copy Project URL và anon/public key
💡 URL có dạng: https://xxxxx.supabase.co (không có dấu / cuối)
💡 Key là anon/public key (không phải service_role key)
2
Tạo bảng dữ liệu (chỉ làm 1 lần)
Vào SQL Editor trong Supabase Dashboard, paste đoạn SQL sau rồi nhấn Run:
-- Từ vựng CREATE TABLE words ( id BIGSERIAL PRIMARY KEY, word TEXT NOT NULL, phonetic TEXT, part_of_speech TEXT, meaning TEXT NOT NULL, example TEXT, topic TEXT DEFAULT 'general', level INT DEFAULT 3, created_at TIMESTAMPTZ DEFAULT now() ); ALTER TABLE words ENABLE ROW LEVEL SECURITY; CREATE POLICY "public_all" ON words FOR ALL USING (true) WITH CHECK (true); -- Câu ngữ pháp CREATE TABLE sentences ( id BIGSERIAL PRIMARY KEY, viet TEXT NOT NULL, eng TEXT NOT NULL, note TEXT, type TEXT DEFAULT 'daily', created_at TIMESTAMPTZ DEFAULT now() ); ALTER TABLE sentences ENABLE ROW LEVEL SECURITY; CREATE POLICY "public_all" ON sentences FOR ALL USING (true) WITH CHECK (true); -- Word Families CREATE TABLE word_families ( id BIGSERIAL PRIMARY KEY, root TEXT NOT NULL, word_ids JSONB, created_at TIMESTAMPTZ DEFAULT now() ); ALTER TABLE word_families ENABLE ROW LEVEL SECURITY; CREATE POLICY "public_all" ON word_families FOR ALL USING (true) WITH CHECK (true); -- Chủ đề tùy chỉnh CREATE TABLE topics ( id BIGSERIAL PRIMARY KEY, key TEXT UNIQUE NOT NULL, label TEXT NOT NULL ); ALTER TABLE topics ENABLE ROW LEVEL SECURITY; CREATE POLICY "public_all" ON topics FOR ALL USING (true) WITH CHECK (true); -- Meta (tiến độ, goal, hard words, families - đồng bộ tự động) CREATE TABLE user_meta ( key TEXT PRIMARY KEY, value TEXT ); ALTER TABLE user_meta ENABLE ROW LEVEL SECURITY; CREATE POLICY "public_all" ON user_meta FOR ALL USING (true) WITH CHECK (true);
3
Đồng bộ toàn bộ dữ liệu lên Supabase
Nhấn nút bên dưới để đẩy tất cả dữ liệu hiện có (từ vựng, câu mẫu, Word Family, chủ đề) lên Supabase:
4
Tải dữ liệu từ Supabase về
Dùng khi bạn đổi thiết bị hoặc muốn load data từ cloud về trình duyệt mới:
✓
Hướng dẫn đồng bộ đa thiết bị
Bước 1 — Lần đầu (chỉ làm 1 lần):
  ① Nhập URL + Key phía trên → "Kiểm tra & kết nối"
  ② Vào Supabase → SQL Editor → paste SQL ở bước 2 → Run
  ③ Về app → Home → nhấn "↑ Lưu lên cloud"

Bước 2 — Thiết bị thứ 2:
  ① Mở app → Supabase tab → nhập cùng URL + Key → kết nối
  ② Home → nhấn "↓ Tải về máy này"

Hàng ngày — dùng 2 thiết bị:
  • Học xong trên thiết bị nào → nhấn "↑ Lưu lên cloud"
  • Mở thiết bị kia → nhấn "↓ Tải về máy này"

⚠ Lưu ý quan trọng:
  • Phải chạy SQL ở bước 2 trước khi dùng
  • Nếu thấy thông báo đỏ khi tải về → kiểm tra Console (F12) để xem lỗi chi tiết
  • Streak/XP đồng bộ được — chỉ cần upload từ thiết bị đang học
